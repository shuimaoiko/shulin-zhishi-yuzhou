## 0. 一句话定位

> **静林档案台 / Quiet Grove Archive** —— 深色"知识档案馆/索引工作台"，不是星图主题页。
> 黑底不是为了"暗"，而是让 1256 条内容像展品一样被照亮。内容优先，装饰退后。

---

## 1. 参考案例（Awwwards · 知识档案 / 数字博物馆类）

| 案例 | 类型 | 抽取的特征 |
|---|---|---|
| **The Museum of the World**（大英博物馆 × Google Cultural Institute） | 数字博物馆 / 交互时间线 / 文化教育 | 黑底 + 极少色（黑 `#000` / 蓝 `#2779a7` / 青 `#49c5b6`）；色彩只服务"内容识别"；空间化的时间/关系导航 |
| **The Graphic Design Archive** | 平面设计史档案 / Minimal / Typography | 近乎单色、留白充足、**字体主导层级**；策展式分章；内容前置、不靠特效 |
| **Built for Archives** | 档案型作品集 / Grid | 黑白单色、网格布局、大量留白让元素呼吸；克制的微交互（hover / 过渡 / 载入）撑起体验 |

### 三者的共同设计特征（本规范的根基）

1. **极度克制的配色**：近单色基底 + ≤3 个"功能色"。色彩只承担三类职责——导航识别、当前状态、重点行动。
2. **字体主导的编辑级层级**：标题自信、正文克制，靠字号/字重/行距拉开节奏，而不是靠颜色和框。
3. **充足留白 / 画廊式取景**：内容四周留出呼吸空间，密集数据也要有节奏，不堆满。
4. **结构化网格 + 探索式导航**：集合用网格/时间线/层级树组织，可横向策展式浏览。
5. **微交互而非大动效**：hover、过渡、聚焦、展开即可；**不做持续自转/大面积背景动画**。
6. **内容前置**：任何装饰都不得与档案内容抢视觉。

## 2. 设计原则（落到本项目）

- 页面优先级：**搜索/索引 > 导图探索 > 列表阅读 > 详情引用**。
- 色彩分工固定：**青 = 当前/主操作/焦点**，**琥珀 = 档案暖意/强调/高亮**，**珊瑚 = 罕见的提醒/事件**，其余一律中性灰。
- 默认完整展示四大系统，用户进来不用找路。
- 动效只用于：进入、聚焦、展开、hover；必须可被 `prefers-reduced-motion` 关闭。
- 不再使用：星空/星点、宇宙发光、彩虹分类色、蓝紫(periwinkle)发光。

## 3. 配色（Color Tokens）

### 基底 / 表面（冷调近黑墨，比纯蓝黑略暖中性）
| Token | 值 | 用途 |
|---|---|---|
| `--bg` | `#0a0d12` | 画布 / 全局底 |
| `--ink` | `#070a0e` | 最深处、渐变收尾 |
| `--panel` | `#12161d` | 一级面板（左栏、卡片） |
| `--panel2` | `#171c25` | 二级面板 / 控件底 |
| `--elev` | `#1c222c` | 浮层 / 弹窗 / 详情 |

### 文字（不用纯白，降低深色上的眩光）
| Token | 值 | 用途 |
|---|---|---|
| `--txt` | `#f3f5f8` | 主文字、标题 |
| `--dim` | `#bcc4ce` | 次文字、正文摘录 |
| `--muted` | `#838d99` | 弱文字、元数据、占位符 |

### 描边 / 分隔（中性冷灰，低透明度发丝线）
| Token | 值 | 用途 |
|---|---|---|
| `--line` | `#283039` | 实色描边 |
| `--hair` | `rgba(236,240,245,.10)` | 标准发丝分隔线 |
| `--soft` | `rgba(236,240,245,.05)` | 极弱内描边 |
| `--rule-strong` | `rgba(236,240,245,.16)` | 强调分隔（标题下） |

### 功能色（≤3，只用于导航识别 / 当前状态 / 重点行动）
| Token | 值 | rgb | 角色 |
|---|---|---|---|
| `--accent`（青·primary） | `#4fb9a9` | `79,185,169` | 当前/选中、焦点环、主操作、链接、关键数字 |
| `--accent2`（琥珀·secondary） | `#d8b367` | `216,179,103` | 档案暖意、高亮/mark、强调、次操作 |
| `--warn`（珊瑚·alert） | `#e98c84` | `233,140,132` | 罕见的提醒/事件，慎用 |

- 青软填充：`rgba(79,185,169,α)`；琥珀软填充：`rgba(216,179,103,α)`。
- 焦点环统一：`box-shadow:0 0 0 3px rgba(79,185,169,.20)`，描边 `rgba(79,185,169,.55)`。

### 系统识别色（4 系统 + 子号，去饱和"陈年档案"色族，各自可辨但同属一家）
| 系统 | 值 | 说明 |
|---|---|---|
| 📕 知乎 | `#7e85ad` | 静谧靛蓝灰 |
| 📗 公众号 | `#5fa78f` | 林雾鼠尾草绿 |
| 📡 B站动态 | `#5f9fc4` | 暗调青蓝 |
| 🎬 B站视频 | `#c9a25c` | 陶土琥珀 |
| └ 对木同学（子号） | `#a07f54` | 档案棕 |

- 分组（如知乎 13 维度）的彩虹色在运行时统一**去饱和 + 向所属系统色混合 ~40%**，保留弱辨识、消除霓虹感。原始 `data.js` 不改（生成物）。

## 4. 字体（Typography）

- 字体栈（保持系统中文优先，免加载、跨端稳定）：
  `-apple-system, BlinkMacSystemFont, "SF Pro Text", "PingFang SC", "Microsoft YaHei", system-ui, sans-serif`
- 数字统一 `font-variant-numeric: tabular-nums`（已全局开启）。
- **字号阶梯**（px）：`11 / 12 / 13 / 14 / 16 / 18 / 22`
  - 22 = 品牌名/区块大标题（display）
  - 18 = 二级标题（H2、列表标题）
  - 16 = 强调正文 / 详情正文
  - 14 = 默认正文 / 搜索框
  - 13 = 列表摘录 / 控件
  - 12 = 元数据 / 按钮 / 标签
  - 11 = 角标 / 段眉（caps）
- **字重**：`400` 常规 / `560` 中 / `700` 粗 / `780` 重（标题）。
- **行距**：标题 `1.18`，正文 `1.5`，密集元数据 `1.4`。
- 段眉/系统标签可用 `letter-spacing:.04em; text-transform:uppercase`（仅限英文/数字短标签）。

## 5. 间距（Spacing · 4px 基准栅格）

| Token | 值 | 典型用途 |
|---|---|---|
| `--sp-1` | `4px` | 图标-文字、紧贴 |
| `--sp-2` | `8px` | 控件内元素间距 |
| `--sp-3` | `12px` | 卡片内边距、行间距 |
| `--sp-4` | `16px` | 区块间距 |
| `--sp-5` | `20px` | 面板内边距 |
| `--sp-6` | `24px` | 大区块/列表外边距 |
| `--sp-7` | `32px` | 页面级留白 |

- 列表阅读区水平内边距 `≥ 24px`；卡片内边距 `12–18px`。
- 留白优先：宁可多留呼吸，不堆满。

## 6. 圆角（Radius）

| Token | 值 | 用途 |
|---|---|---|
| `--r-sm` | `8px` | 标签/输入/小按钮 |
| `--r-md` | `10px` | 面板/卡片/主按钮 |
| `--r-lg` | `14px` | 弹窗/详情/大卡片 |
| `--r-pill` | `999px` | 胶囊/圆点/小标记 |

- 收敛圆角种类，避免 9/11/13/16 混用造成的杂乱；新写组件只从上表取值。

## 7. 布局（Layout）

- **三段式工作台外壳**：
  - 顶部 = **命令区**（command bar）：品牌 + 搜索（主入口）+ 视图/呈现切换 + 索引按钮；统计做紧凑徽章。
  - 左侧 = **来源导航**（collection navigator，约 `280px`）：系统标题为一级、分组为二级；可折叠。
  - 主区 = **画布 / 列表**：导图探索为大留白画布，列表阅读为居中阅读流。
- 列表阅读最大宽度 `≈ 1100–1180px` 居中。
- 画布保持大面积留白，连线看得清但不抢文字标签。
- 响应式：`≤980px` 顶栏堆叠、左栏收窄；`≤700px` 进一步压缩，隐藏次要徽章。

## 8. 动效（Motion）

- **时长**：微交互 `140–220ms`；面板/浮层进入 `280–420ms`。
- **缓动**：`cubic-bezier(.2,.8,.2,1)`。
- **允许的动作**：进入（轻微上移淡入）、hover（边框/底色/左侧细线变化，**不大幅缩放**）、聚焦、展开、导图高亮（高亮当前、其余暗调，不用模糊）。
- **禁止**：持续自转、大面积背景动画、高成本滤镜动画。
- 全部包裹在 `@media (prefers-reduced-motion:no-preference)` 内，或可被其关闭。

## 9. 组件规则（Component Rules）

- **顶部命令区**：搜索框是主入口（最高视觉权重）；切换器用分段按钮；统计只做小徽章。
- **左侧来源导航**：系统标题带左侧 `3px` 系统色竖条；分组用小圆点系统色；hover 仅轻微位移 + 提亮。
- **导图画布**：节点色取系统/分组识别色（已去饱和），连线中性灰；高亮聚焦时其余暗调。
- **列表条目**：每条是一份档案条目；hover 时左侧出现青→琥珀细条 + 当前条提亮，其余条不动。
- **索引**：独立弹窗，不侵占左侧导航。
- **详情**：正文高对比，元数据清楚但不抢标题；引用块左侧琥珀细条。
- **mark 高亮**：琥珀底 `rgba(216,179,103,.30)` + 浅琥珀字。
- **按钮**：主操作=青描边/青焦点；次操作=中性；危险/提醒才用珊瑚。

## 10. 治理与迭代（怎么用这份规范）

本文件是审美决策的"单一事实源"。三种迭代姿势：

1. **喜欢某个东西** → "我看到一个网站/截图，分析它的风格，把好的部分更新进 DESIGN.md，然后应用。"
2. **觉得哪里不对** → "首页标题区太挤了，调整一下，更新 DESIGN.md。"
3. **大改风格** → "重新找 3 个参考，重建 DESIGN.md。"

原则：**所有审美决策先进 DESIGN.md，再落到代码**；每次判断自动沉淀，不丢、不用重复说。
