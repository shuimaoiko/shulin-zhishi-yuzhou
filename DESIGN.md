## 0. 一句话定位

> **静林档案台 / Quiet Grove Archive** —— 深色"知识档案馆 / 索引工作台"，不是星图主题页。

## 1. 参考案例与共性（Awwwards · 知识档案 / 数字博物馆类）

| 案例 | 视觉抽取 | 交互 / 内容抽取 |
|---|---|---|
| **The Museum of the World**（大英博物馆 × Google） | 黑底 + 极少色（黑/蓝/青 `#49c5b6`）；色只服务内容识别 | 时间轴主导导航；discover → 探索展品 → 详情；每件展品是时间框架里的离散可探索条目；流畅 HTML5 微交互 |
| **The Graphic Design Archive** | 近单色、留白足、**字体主导层级** | **分章式导航**（按时期分段）；**多维分类**（按运动/学科/技法）= 标签筛选；**渐进式信息披露**；路径：浏览章节 → 标签筛选 → 看条目详情 |
| **Built for Archives** | 黑白单色网格、大量留白 | **多级导航**展开子类；清晰层级（概览/要素/细节）；**hover/loading/page-transition 作为一等微交互**；内容=带缩略图的要素卡 + 功能筛选 |

### 共性（本规范的根基）
- **视觉**：极度克制配色（近单色 + ≤3 功能色）/ 字体主导层级 / 充足留白 / 内容前置。
- **交互**：渐进式披露（概览露出、深度下钻）/ 多维分类筛选 / 结构化导航 / 主入口显眼、次控件收纳。
- **微交互**：hover、loading、切换、滚动被当作一等细节专门设计，而非附带。
- **内容**：每条内容是离散可探索的"档案条目"，列表/卡片 → 详情/引用 一以贯之。

## 2. 设计原则

1. 页面优先级恒定：**搜索/索引 > 导图探索 > 列表阅读 > 详情引用**。
2. 色彩分工固定：**青=当前/焦点/主操作**，**琥珀=档案暖意/强调/高亮**，**珊瑚=罕见提醒**，其余中性。
3. 默认完整展示四大系统，用户进来不用找路；深层细节默认折叠，按需展开。
4. 动效只用于：进入、聚焦、展开、hover、切换；**不做持续自转/大面积背景动画**；可被 `prefers-reduced-motion` 关闭。
5. 不再使用：星空/星点、宇宙发光、彩虹分类色、蓝紫(periwinkle)发光。
6. 只归纳分类，不提炼总结——呈现层不替用户下结论，只把内容组织清楚、可检索、可引用。
7. **清爽优先（少即是多）**：界面材质极简——实色面板 + 发丝描边 + 极简阴影，装饰归零，留白为王。每加一个元素/渐变/阴影前先问"删得掉吗"，删不掉再问"能更轻吗"。

## 4. 视觉风格（Visual）

### 4.1 配色 Tokens
**基底/表面**（冷调近黑墨，比纯蓝黑略暖中性）
| Token | 值 | 用途 |
|---|---|---|
| `--bg` | `#0a0d12` | 画布/全局底 |
| `--ink` | `#070a0e` | 最深处、渐变收尾 |
| `--panel` | `#12161d` | 一级面板（左栏、卡片） |
| `--panel2` | `#171c25` | 二级面板/控件底 |
| `--elev` | `#1c222c` | 浮层/弹窗/详情 |

**文字**（不用纯白，降低深色眩光）：`--txt #f3f5f8` 主 / `--dim #bcc4ce` 次 / `--muted #838d99` 弱。
**描边**：`--line #283039` 实色 / `--hair rgba(236,240,245,.10)` 发丝 / `--soft .05` 极弱 / `--rule-strong .16` 强调。

**功能色（≤3）**
| Token | 值 | rgb | 角色 |
|---|---|---|---|
| `--accent`（青） | `#4fb9a9` | `79,185,169` | 当前/选中、焦点环、主操作、链接、关键数字 |
| `--accent2`（琥珀） | `#d8b367` | `216,179,103` | 档案暖意、高亮/mark、强调、次操作 |
| `--warn`（珊瑚） | `#e98c84` | `233,140,132` | 罕见提醒/事件，慎用 |

**焦点环统一**：`box-shadow:0 0 0 3px rgba(79,185,169,.20)` + 描边 `rgba(79,185,169,.55)`。

**系统识别色**（去饱和"陈年档案"色族，各自可辨同属一家）：知乎 `#7e85ad` / 公众号 `#5fa78f` / B站动态 `#5f9fc4` / B站视频 `#c9a25c` / 对木同学子号 `#a07f54`。
- **原因**：参考案例都用"近单色 + 极少功能色"，色彩只服务识别。三功能色对应"当前/暖意/提醒"三种职责，超过就会互相抢、失去信号意义。分组彩虹色在**运行时**去饱和 + 向系统色混合 ~40%（不动生成物 `data.js`），保留弱辨识、消除霓虹。

### 4.2 间距 / 圆角 / 材质
- **间距**（4px 基准）：`--sp-1:4` `--sp-2:8` `--sp-3:12` `--sp-4:16` `--sp-5:20` `--sp-6:24` `--sp-7:32`。列表阅读区水平内边距 `≥24px`，卡片内边距 `12–18px`。
- **圆角**：`--r-sm:8`(标签/输入/小按钮) `--r-md:10`(面板/卡片/主按钮) `--r-lg:14`(弹窗/详情/大卡片) `--r-pill:999`(胶囊/圆点)。新组件只从这四档取值。
- **材质（清爽基线 v3）**：面板/卡片/控件一律**实色**（`--panel` / `--panel2` / `--elev`，或 `rgba(255,255,255,.025–.05)` 微提层），**发丝描边 `--hair`**；阴影只在**浮层**(弹窗/详情/缩放控件)用 `--shadow` / `--shadow-soft`，**常驻面板（顶栏/左栏/列表卡）一律无阴影**；浮层 `backdrop-filter ≤ blur(16px)`。
- **禁用的纯装饰**：多色角向背景光晕、星点噪点、`mix-blend` 叠加层、标题渐变下划线（`::after` 金青条）、按钮/卡片多层渐变底、密集网格（画布只留极弱单层 `128px` 网格）。**系统色竖条 / 圆点保留**（功能性 wayfinding，用实色）。
- **hover/active 反馈**：用底色微提（`rgba(255,255,255,.05)`）或低饱和青/琥珀填充，**不再用渐变+厚投影**。
- **原因**：参考案例靠"近单色 + 留白"立秩序；渐变、发光、多层投影会让深色界面显得"脏"且拥挤，元素互相抢。清爽 = 表面统一、装饰归零、对比交给排版与功能色、空间交给留白。收敛间距/圆角档位（避免 9/11/13/16 混用）也是同一目的。

## 5. 字体体系（Typography）

### 5.1 字体栈与中英文搭配
- 统一栈：`-apple-system, BlinkMacSystemFont, "SF Pro Text", "PingFang SC", "Microsoft YaHei", system-ui, sans-serif`。
- **中英文搭配规则**：中文走 PingFang SC / 思源黑体（系统兜底）；英文与数字走 SF Pro / system sans（在同一栈里自动按字符落字）。**数字一律 `font-variant-numeric:tabular-nums`**（已全局开启），保证统计/计数对齐。
- 不引入需要联网加载的 Web Font；不给中文用斜体（中文斜体是机械倾斜，难看）。
- **原因**：免加载、跨端稳定、首屏不抖；中英文各取所长，数字等宽便于"1256 条""103/6/221"这类计数纵向对齐。

### 5.2 字号阶梯与层级对比
| 级别 | 字号 | 字重 | 行高 | 用途 |
|---|---|---|---|---|
| Display | `22px` | 780 | 1.18 | 品牌名 / 区块大标题 |
| H2 | `18px` | 780 | 1.22 | 列表标题、弹窗标题、详情标题 |
| 强调正文 | `16px` | 400/700 | 1.5 | 详情正文、重点内容 |
| 正文 | `14px` | 400 | 1.5 | 默认正文、搜索框 |
| 摘录/控件 | `13px` | 400/560 | 1.5 | 列表摘录、下拉/按钮文本 |
| 元数据 | `12px` | 560 | 1.4 | 元信息、按钮、标签 |
| 角标/段眉 | `11px` | 700 | 1.4 | 计数角标、英文段眉(caps) |

- **层级对比**：相邻级别至少差 1 档字号**或** 1 档字重，不靠颜色拉层级（颜色留给功能色）。标题与正文对比靠"字号 + 字重 + 行高"三者叠加，而非加粗一项。
- **引用 / 原文摘录**：字号降一档（13px）、色用 `--dim`、左侧 `3px` 琥珀竖条、行高 1.6、可加 `--panel2` 略深底——与正文明显区分但不喧宾夺主。
- **原因**：参考案例都是"字体主导层级"。深色界面上颜色对比已被功能色占用，层级必须由排版承担；引用是"别人的话/原文"，要可辨识可引用，故用竖条+缩进+降级处理。

### 5.3 字重节奏
- 可用字重：`400` 常规 / `560` 中(标签、控件) / `700` 粗(强调) / `780` 重(标题)。
- **同屏可见字重 ≤3 种**；正文区域避免 700+ 大段（仅用于行内强调）。
- 段眉/英文短标签可 `letter-spacing:.04em; text-transform:uppercase`，仅限英文/数字。
- **原因**：字重过多会让页面"嘈杂"，丧失节奏；克制的字重对比才有编辑级秩序感。

### 5.4 行高与段距（呼吸感）
- 行高：标题 `1.18`，正文 `1.5`，密集元数据 `1.4`，引用 `1.6`。
- 段距：列表条目间 `6–8px`；条目内标题↔摘录 `5–6px`；区块之间用 `--sp-4/6`。
- 单行文本（标题、标签）用 `white-space:nowrap` + 必要时 `text-overflow:ellipsis`，避免折行破坏层级。
- **原因**：深色长文本更需要行距撑开可读性；统一段距让密集的 1256 条仍有节奏，不至于糊成一团。

## 6. 交互规范（Interaction）

### 6.1 信息架构与导航结构
- **三段式工作台外壳**（固定）：
  - **顶部 = 命令区**：品牌+统计徽章（左）｜搜索（中，**主入口，视觉权重最高**）｜视图/呈现切换 + 索引按钮（右）。
  - **左侧 = 来源导航**（约 `280px`）：系统标题为一级（带 `3px` 系统色竖条）、分组为二级（小圆点系统色）；可折叠。
  - **主区 = 画布 / 列表**：默认进**思维导图**（探索视图，大留白画布）；可切**列表**（阅读流，居中 `≤1180px`）。
- **多维呈现**：顶部"导图呈现"切换内容轴——公众号支持「类型+时间 / 类型」，对应参考案例的"多维分类(时期/学科/技法)"。知乎无逐条时间，故只给「回答/文章/想法」内容类型，不伪造时间。
- **原因**：外壳恒定 = 用户的"肌肉记忆"不被打断；左导航是 collection navigator，顶部是 command center，主区是展台，三者职责分明。

### 6.2 功能收纳：露出 vs 折叠
| 露出（一级，常驻） | 折叠/收纳（按需） |
|---|---|
| 搜索框（顶部中央，最大） | 索引 → **独立弹窗**（不占左导航） |
| 四大系统 + 计数（左栏） | 知乎分组**默认收起**（条目最多，先收） |
| 视图切换 / 导图呈现 | 帮助 `?`、重置 `⟲` → 右下角**小图标按钮** |
| 缩放控件（画布右下） | 单节点 >28 条 **自动分批**（每组 ≤24，点开才展开） |
| 当前筛选结果计数 | 详情 `#detail` → 点条目/节点才滑出 |

- **规则**：一屏内常驻控件控制在"搜索 + 系统导航 + 视图切换 + 缩放"四组以内；其余进弹窗/角落/折叠/分批。新功能优先问"能不能折叠或进弹窗"，默认不占顶部一级。
- **浮于画布上的控件（缩放/组织/展开折叠/帮助）规则**：静默时**半透明 `opacity:.62`**、`hover/focus` 才实体化为 `1`——让它们不与下方节点内容争视觉、不形成"压在内容上的大方块"。
- **浮层工具条按职责分组**：同一浮层里把控件分成「**组织**(导图内容轴) ｜ **动作**(展开/折叠) ｜ **工具**(重置/帮助)」三组；动作用文字钮(flex 等宽)，工具用**图标钮**(透明底、muted 色、固定宽)，两组间加 `1px` 发丝**分隔线**。原因：一锅烩的按钮（文字+符号混排、无边界）让人分不清哪些是"改内容"、哪些是"操作画布"、哪些是"辅助"；分组 + 按钮形态差异让职责一眼可分。
- **原因**：渐进式披露是三个参考案例的共同手法——概览清爽、深度按需。1256 条若全展开会文字重叠、操作卡顿（已有教训），故大节点分批、深层默认折叠。浮层控件半透明则是"清爽优先"（§2.7）在画布上的落地：需要时清晰、不需要时退场。

### 6.3 按钮位置与信息层级
- **位置约定**：主操作（搜索/索引）在顶部；筛选在左栏；画布操作（缩放/重置）贴画布右下；条目级操作（导出/查看原文）在条目或详情内就近放置。
- **层级（视觉权重从高到低）**：搜索框 > 主按钮(青描边/青焦点) > 分段切换器 > 次按钮(中性) > 角落图标按钮。危险/提醒态才用珊瑚。
- **导出语义**：无搜索词时 `导出 MD` = 当前左侧筛选结果；有搜索词时按钮变 `导出关键词` = 全库命中该词，不受左侧开关影响。按钮文案随状态变化以表达作用域。
- **命名与按钮区分（强制）**：并列/相邻的控件名称**必须互相可区分**——不共享词根、不靠细微差别（如曾经的"类型时序"vs"类型"）。用**不同概念词**命名各档，并配 `title` 说明各自作用。分段控件的标签要划清职责边界：
  - `视图`（导图 / 列表）＝**看哪里**；`组织`（时间序 / 分组 / 合并来源）＝导图内容**怎么排**。两者 `title` 不得都写"呈现/呈现方式"。
  - `组织` 三档：**时间序**＝按发布时间；**分组**＝按账号/维度细分；**合并来源**＝跨账号合并只按类型。
- **原因**：位置即语义——操作放在它作用的对象附近，用户不用猜；权重阶梯让眼睛先看到主路径；名称互斥 + 边界清晰，用户才不会在两个长得像的控件间犹豫。

### 6.4 用户操作路径
- **主路径**：进入(见四系统总览) → 搜索/点系统 → 在导图里探索/下钻 → 点条目看详情 → 复制原文/导出引用。
- **检索路径**：顶部搜索（匹配标题/摘录/账号/类型/系统分类/标签，已防抖）→ 专用索引结果树（系统→类型→账号→分页组，每页 ≤24，点分页组才展开具体内容）。
- **索引路径**：顶部「索引」开弹窗 → 输入任意关键词"查看导图"或"导出关键词"；或在滚动索引词列表里对已有词直接"查看/导出"。
- **机读路径**：`agent.html?q=关键词&platform=all&format=md|json`、`id=agent_id` 调单条——给 Agent 用，不改前端导出。
- **状态切换规则**：搜索态↔普通导图切换用**硬清图**（取消待恢复视口/分页/缩放帧并清旧图层），避免关键词线条残留。
- **原因**：路径对应优先级"搜索/索引>探索>阅读>引用"；专用结果树避免按全量叶子布局散开；硬清图是历史踩坑后的必要约束。

## 7. 微交互（Micro-interactions）

> 时长：微交互 `140–220ms`；面板/浮层进入 `280–420ms`。缓动统一 `cubic-bezier(.2,.8,.2,1)`。

### 7.1 悬停 Hover
- 卡片/条目：边框提亮 + 底色微提 + **左侧出现青→琥珀细条**；**不大幅缩放**（最多 `translateY(-1px)` 或 `translateX(2px)`）。
- 按钮：底色/边框变化 + `translateY(-1px)`；图标按钮仅变色。
- 导图节点：hover 高亮当前、其余**轻微暗调**，不模糊。
- **原因**：hover 要给"可点"信号但不能跳动扰乱阅读；左侧细条是档案条目的"翻页指示"，比整卡放大更克制。

### 7.2 点击 / 激活反馈
- 按钮 `:active` 轻微下沉 `scale(.98)`；分段切换 active 态用青/琥珀低饱和填充 + 内描边。
- 选中条目/节点：保留高亮态（`focus-row`/节点高亮）直到切换，让用户知道"我在看哪条"。
- **原因**：即时物理反馈确认"点到了"；持久选中态维持上下文。

### 7.3 页面 / 视图切换
- 面板/列表进入：`fadeUp`（透明+上移 8–10px，280–340ms）。
- 导图↔列表、搜索态↔普通态：先硬清旧图层再渲染新内容，避免残影；切换不重置用户视口（展开/收起节点时**保存并恢复当前视口**，不强制居中）。
- **原因**：进入动画给"内容来了"的节奏；视口保持避免打断"用户正在看的位置"（历史踩坑）。

### 7.4 滚动 / 缩放 / 平移
- 画布缩放：`#chart` 内自管 wheel，`preventDefault()` 阻止浏览器页面缩放/滚动，按指针位置缩放当前 series 视图组；手感参数 `ZOOM_SENSITIVITY=0.00082`、单帧 `ZOOM_DELTA_CAP=160`（防触控板一把窜飞）。
- 平移：tree `roam:'scale'`，平移由自定义全局拖动接管，拖空白或标签区都能移动视图组。
- 列表滚动：`overflow:auto` + `overscroll-behavior:contain`，不连带滚动外层。
- **原因**：浏览器页面缩放常来自 ctrlKey/触控板 pinch 的默认 wheel；只要在画布内缩放就必须阻止默认（核心教训）。参数是多轮调出的手感，改动须真机验证。

### 7.5 加载 / 空 / 错误状态
- 加载：数据较大(~2.2MB)，首屏给轻提示而非空白；echarts 渲染期间画布保持底纹，不闪烁。
- 空结果：搜索无命中时给"无匹配"占位文案 + 清除入口，不留空白画布。
- 离线：`agent-index.js`/`data.js` 支持离线读取（GitHub Pages / 双击本地）。
- **原因**：克制不等于无反馈；状态缺失会让用户以为"卡了/坏了"。

### 7.6 动效预算 & 可达性
- 禁止：持续自转、大面积背景动画、高成本滤镜动画（会拖慢交互——已两次踩坑）。
- 所有非必要动效包在 `@media (prefers-reduced-motion:no-preference)` 内，或可被其关闭。
- **原因**：性能与可达性优先；动效是调味不是主菜。

## 8. 内容呈现（Content）

### 8.1 列表条目（档案条目卡）
- 结构：`grid-template-columns:minmax(0,1fr) auto`——左为标题+摘录+元数据，右为类型 chip。
- 层级：标题(15px/700) → 摘录(13px/`--dim`，截断) → 元数据(12px/`--muted`：系统·账号·类型·时间)。
- hover：左侧青→琥珀细条 + 当前条提亮，其余条不动；条目可点进详情。
- 间距：条目间 `6–8px`，内边距 `13–18px`，圆角 `--r-sm/md`。
- **原因**：每条是离散"档案条目"（参考案例的核心模型）；一行内信息按"是什么→讲什么→出处"排序，右置 chip 便于扫读分类。

### 8.2 卡片 / 导图节点
- 节点色取系统/分组识别色（已去饱和），连线中性灰 `rgba(126,146,172,.46)`，曲线弱化 `curveness:.20`。
- 中心/根节点用青 `#4fb9a9`；分页组节点命名「第1组 · 1-24」（时间视图附日期范围）。
- 标签：`width` 截断 + `ellipsis`，避免重叠；高亮聚焦时其余暗调。
- **原因**：节点是可探索条目的空间化表达；连线中性、识别色克制，保证"看得清但不抢文字"。

### 8.3 详情页结构（阅读 + 引用视图）
- 滑出面板 `#detail`：标题(H2) → 元数据徽章(系统/账号/类型/时间) → 正文区(16px 高对比) → 原文链接/导出。
- 引用块 `.ex`：左侧琥珀 `3px` 竖条 + 略深底 + 行高 1.6，标明"这是原文摘录"。
- 徽章用琥珀实底深字（高优先识别），元数据清楚但不抢标题。
- **原因**：详情承担"阅读 + 引用"双职责；引用块视觉独立，方便用户判断哪些是可直接引用的原文。

### 8.4 标签与分类展示（多维 taxonomy）
- 维度顺序固定：**系统 → 分组/账号 → 类型 → 时间**（与导出 MD 结构一致）。
- 标签视觉：系统=色竖条/色点 + emoji；分组=小圆点 + 名称 + 计数；类型=chip(系统色描边)。
- 计数常驻在标签右侧（tabular-nums 对齐），让"有多少"一眼可见。
- emoji 仅作辨识前缀，不承担色彩职责（色彩归系统识别色）。
- **原因**：多维分类是档案站的灵魂（三参考都用）；固定维度顺序让"导图→列表→详情→导出"心智一致，计数提供规模感。

### 8.5 高亮 / 搜索命中 / 导出
- 命中高亮 `mark`：琥珀底 `rgba(216,179,103,.30)` + 浅琥珀字；标题与摘录内都高亮。
- 搜索结果树：系统→类型→账号→分页组(≤24)，点分页组才展开，避免全量叶子散开。
- 导出 MD 结构：系统→分组/账号→类型→条目，每条带元数据+正文摘录+原文链接。
- **原因**：高亮用琥珀(强调色)而非青(操作色)，语义不混；导出结构镜像浏览结构，所见即所得。

## 10. 治理与迭代（怎么用这份规范）

本文件是体验决策的"单一事实源"。三种迭代姿势：
1. **喜欢某个东西** → "我看到一个网站/截图，分析它的风格，把好的部分更新进 DESIGN.md，然后应用。"
2. **觉得哪里不对** → "首页标题区太挤了，调整一下，更新 DESIGN.md。"
