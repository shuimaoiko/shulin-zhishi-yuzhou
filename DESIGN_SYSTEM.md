# 设计规范 · 树林知识宇宙

> 目标：把页面从“功能型深色单页”升级为“档案馆级知识工作台”。审美参考 Awwwards 上的知识探索、数据可视化和数字博物馆案例，执行时保持低干扰、可长期阅读、不卡顿。

## 参考案例

1. Digital Curator
   - 类型：艺术集合探索 / 数据可视化 / Gallery / Storytelling。
   - 可借鉴：极少色彩、清晰过滤入口、内容集合的“策展感”。

2. Histography
   - 类型：交互时间线 / 文化教育 / WebGL 数据可视化。
   - 可借鉴：黑底大画布、少量高识别色、节点与时间/关系的空间化表达。

3. The digital Van Gogh Museum
   - 类型：数字博物馆 / 文化教育 / Clean / Horizontal Layout / Storytelling。
   - 可借鉴：黑白金三色、横向叙事、内容优先而不是装饰优先。

## 设计原则

- 黑底不是“暗”，而是让内容像展品一样被照亮。
- 色彩只承担三类职责：导航识别、当前状态、重点行动。
- 页面优先级：搜索/索引 > 导图探索 > 列表阅读 > 详情引用。
- 动效只用于进入、聚焦、展开、hover，不做持续自转或高成本背景动画。
- 默认状态要完整展示四大系统，避免用户一进来就需要找路。

## 视觉令牌

- 基底：`#070A10` / `#0A0F17`
- 面板：`rgba(15,23,34,.88)` / `rgba(22,31,45,.94)`
- 主文字：`#FBFCFF`
- 次文字：`#C8D2E0`
- 弱文字：`#9EAABC`
- 时间线青：`#49C5B6`
- 博物馆金：`#ECD06F`
- 策展棕：`#987654`
- 事件珊瑚：`#FF9398`

## 组件规则

- 顶部：像 command center，搜索框是主入口，统计只做紧凑徽章。
- 左侧：像 collection navigator，系统标题为一级，分组为二级。
- 导图：画布保持大面积留白，线条要看得清但不能抢文字。
- 列表：每条内容是档案条目，hover 时高亮当前条，其他条轻微弱化。
- 索引：独立弹窗，不侵占左侧导航。
- 详情：正文区域高对比，元数据清楚但不抢标题。

## 交互动效

- 面板进入：150-280ms，轻微上移淡入。
- hover：边框、背景、左侧细线变化，避免大幅缩放。
- 导图高亮：保留节点本身高亮，其他内容暗调，不使用模糊。
- 所有背景动效都应可被 `prefers-reduced-motion` 关闭。
